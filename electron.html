Electron

Electron enables people with a primarily web focused skillset to create applications for a desktop environtment. Well known examples of such apps are Spotify, Slack and Microsoft Teams. This was my first proper attempt at planning and writing my own Javascript code from scratch, so a fun learning experience overall. I decided to create a countdown timer app and then gave the app an identity with a simple branding process and a mock landing page. This was essentially a toungue in cheek exercise, given that there are plenty of better solutions out there covering pretty much the same need.

Input and storage

This project was my first forray into gathering user input and storing it. For the purposes of this app I needed two bits of data; a description of a future occasion and its date. Thanks to storage, this data will be retrieved next time you open the app so the countdown clock will always show the correct remaining time.

Layout

While the amount of information to be displayed is miniscule, I found the number of ways I could present it to be almost endless. User testing led me towards a fairly gentle horizontal presentation, but as Electron allows you to use CSS code to create responsive layouts, I saved the vertical layout option for those users who wanted a smaller version of the app to sit at the edge of their workspace.

Microcopy

Microcopy proved much harder than I had thought, as my desire to create a minimal and elegant app interface increased the scope for misunderstanding. Again I found the answers through user testing, taking care to test the app on a wide range of user demographics.

Brandname and icon

All Electron apps need an icon. This is the icon that will sit on the task bar of your computer when the app is running. I started out exploring both typography and the hour glass motif - but separately. Eventually I realised that by taking a little inspiration from gestalt theory I could combine the two together. A slightly unusual name for an app perhaps, but something that I arrived at after test users encouraged me to abandon more conventional ideas.

Download page

Following in the great tradition of free apps I created a simple landing page, from where one could access both the online and the desktop version of the app. Like the app itself, this was also a mostly tongue in cheek exercise given that the demand for this app is pretty much non-existent. Nevertheless, it was a nice way to round off the project and it gave me a chance to practice responsive webdesign.
